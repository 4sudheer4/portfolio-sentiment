1

Git & GitHub

Version control — your code's home base

~1 day ▼
📁

What to learn

git init, git add, git commit, git push — the four commands you'll use every single day.

🎯

Goal

Push your sentiment analyzer code to a public GitHub repo. This is your portfolio piece.

📚

Resources

GitHub's own docs.github.com quickstart is the best. Takes 30 minutes.

🏷️

Interview value

Every single tech job requires this. Non-negotiable baseline.

2

Docker

Package your app so it runs anywhere

~2 days ▼
🧠

Core concept

A Docker container is like a lunch box — it packs your app + all its dependencies together so it works the same on your laptop, a server, or anyone else's machine.

📄

What to learn

Write a Dockerfile. Learn docker build, docker run, docker ps. Run your Flask app inside a container locally first.

🎯

Goal

Run your sentiment app via docker run -p 5000:5000 sentiment-app and see it open in your browser.

🏷️

Interview value

Asked in almost every backend/DevOps interview. "Explain Docker vs VM" is a classic question.

3

AWS EC2

Rent a server in the cloud

~2 days ▼
🧠

Core concept

EC2 is just a Linux computer you rent from Amazon. The t2.micro instance is free for 12 months — perfect for learning.

📄

What to learn

Launch an EC2 instance. SSH into it with your .pem key. Install Docker on it. Understand Security Groups (firewall rules).

🎯

Goal

SSH into your EC2 instance, pull your GitHub repo, run your Docker container, and access your app via the EC2 public IP on your phone.

🏷️

Interview value

AWS is #1 cloud provider. EC2 knowledge alone puts you ahead of many candidates. Pair with IAM basics for bonus points.

4

Nginx

A proper web server in front of Flask

~1 day ▼
🧠

Core concept

Flask's built-in server is for development only. Nginx sits in front and forwards traffic to your app — this is called a reverse proxy. It also handles HTTPS.

📄

What to learn

Write an Nginx config with proxy_pass. Enable it with sites-enabled. Understand ports 80 vs 443.

🎯

Goal

Access your app via http://YOUR_EC2_IP (no port number) — Nginx routes it cleanly to Flask.

🏷️

Interview value

"What is a reverse proxy?" is asked constantly. Now you've actually configured one.

5

CI/CD with GitHub Actions

Auto-deploy on every git push

~2 days ▼
🧠

Core concept

Every time you do git push, GitHub automatically builds your Docker image, SSHes into EC2, and restarts your app. Zero manual steps.

📄

What to learn

Write a .github/workflows/deploy.yml file. Learn about GitHub Secrets for storing your EC2 credentials safely.

🎯

Goal

Edit a line of code, push to GitHub, wait 2 minutes, and see the change live on your phone — automatically.

🏷️

Interview value

CI/CD is the #1 DevOps concept asked about in interviews. This makes your resume line "built CI/CD pipeline" actually true and explainable.

Total time: ~8 days at a comfortable learning pace  ·  Resume line: Python · Flask · Docker · AWS EC2 · Nginx · GitHub Actions
